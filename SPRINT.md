# Sprint 15 — Milestone 10: Agent-addressable review — shared contract + compliance slice

This Sprint lands the **shared review contract** — a single-source grammar file, a deterministic validator script with a thin agentic interpretation contract, and fixture examples covering all four human-gated review artifact families — and migrates the crispest family, **compliance**, end-to-end onto it: `compliance_report` emits structured review items (a `review-id` anchor plus explicit `Decision:` / `Decision-note:` fields per violation), `compliance_fix` consumes those fields through the shared validator, and a new **`amanuensis-review` Claude Code skill** (installed into consuming projects by `install.sh`) walks a human through pending review units, records their decisions by `review-id`, and reports accurate progress counts. The companion is the human-decision capture layer and progress ledger — never the checker, fixer, or decider. This is the first of four one-milestone-per-sprint slices (M10–M13 in ROADMAP.md); anti-AI, prose-pass, and metaphor migrations follow in later sprints against the contract this Sprint proves. Each slice is a big-bang migration for its family: after this Sprint, the old positional compliance annotation format is an invalid input and no compatibility path is kept.

## Background — what is and isn't wrong today

Established by inspection during planning; tasks should not re-derive this.

- **Each artifact family has its own annotation convention, none addressable or countable.** Compliance uses positional inline tokens — `FIX` / `FIX: [instruction]` / `SKIP` / `ESCALATE` written after violation lines (`agents/steps/compliance-fix.md:37-42`), located by quoted prose anchor (`agents/steps/compliance-fix.md:56`). Prose-pass is the closest to the target model: a per-finding `Annotation:` line with the same token set, declared the single canonical grammar definition (`agents/steps/prose-pass.md:273-290`). Anti-AI adds per-category `BULK:` headers gated by a per-scene `BULK eligibility` declaration (`agents/steps/anti-ai-report.md:49-65`, `:275-307`). Metaphors have no tokens at all: a free-text `Human Assessment:` line (`agents/steps/metaphor-identify.md:72`), where no action word means accepted and **deletion means rejected** (`agents/steps/metaphor-fix.md:40`, `:49`), and bare `REPLACE` is silently treated as `WORKSHOP` (`agents/steps/metaphor-fix.md:69`).
- **Nothing distinguishes accepted from unreviewed at the unit level.** A metaphor entry with no action word and an entry the human never looked at are identical; a compliance report with some annotated entries silently skips the bare ones (`agents/steps/compliance-fix.md:81`). There is no per-unit progress ledger anywhere — reviewing seventy-five metaphors is an open-ended markdown blob, not a bounded queue.
- **The review-state model is already general and does not change.** M9 defined `review_pending` / `reviewed` / `override` (`agents/orchestrator.md:28-33`) and the Artifact-state contract (`agents/orchestrator.md:150-181`); the unannotated-report blocker is uniform across the fix steps (`agents/steps/compliance-fix.md:140`, `agents/steps/prose-fix.md:155`, `agents/steps/anti-ai-fix.md:193`); override is stale-axis-only and anchor-gated (`agents/orchestrator.md:158`). M10 changes none of these mechanics — it changes what the review *evidence* looks like for a migrated family (a filled `Decision:` field) and makes it countable.
- **The entry-point precedent is `revise`:** a thin host adapter over a canonical contract doc (`templates/dispatcher/.claude/commands/revise.md:5` → `agents/revision.md`), installed by `install.sh` (`install.sh:9-24`), explicitly not a pipeline step. The companion follows the same not-a-step rule but ships as a Claude Code **skill** (owner decision) — `install.sh` has no `.claude/skills/` path yet and must learn one.
- **The step set is unchanged.** No step is added or removed and no `step_id` changes, so `scripts/check-pipeline-state.sh`, both CI workflow ymls, and both `pipeline-state.md` files stay byte-for-byte unchanged (the same guarantee Sprints 13–14 relied on).
- **Smoke recipes 3–5 hand-author old-format compliance reports** (`examples/smoke/README.md:103` on) and become invalid specimens once compliance migrates; recipes 6–8 use anti-AI, which keeps its current grammar until M11 and stays untouched.

## Conventions adopted by this Sprint

Locked at planning (starred items are owner decisions from this Sprint's planning session); tasks don't rediscover them.

- **★ One milestone per sprint.** The former monolithic M10 was split into M10 (contract + compliance), M11 (anti-AI), M12 (prose-pass), M13 (metaphors), with reverse ingestion renumbered to M14 — the restructure landed with this plan's commit. Sprint 15 implements M10 only. Rationale: keeps planning and execution 1:1 and each PR reviewable. Rejected alternative: the whole migration in one sprint (~14 tasks; the four migrations would land against a contract never exercised).
- **★ The validator is a deterministic script; agentic judgment interprets it.** `scripts/validate-review-artifact.sh` (read-only POSIX sh, `check-pipeline-state.sh` precedent) owns parse + validate + count; the companion and the fix/apply steps *run* it and act on its ledger and exit code, per the thin interpretation contract in `agents/review-validation.md`. Rationale: the ledger counts and the proceed/block verdict are the two things that must never be wrong, and "run this and honor the exit code" is a harder instruction for an agent to fumble than a prose counting discipline; strictness is a feature — `invalid` is a legal, reported state with line-naming errors, and the companion writes canonical format so hand-edit noise shrinks on the write path. The YAML stays the single grammar source: the script reads its token lists and bulk rules from `agents/review-grammars.yaml`, never hardcodes them. Rejected alternatives: a pure agentic prose procedure (counting by LLM attention is the design's weakest point) and shell-assisted agentic counting (the middle path chosen first at planning, then flipped after discussion — it still left the proceed verdict assembled by prose-following). If the strict script proves too rigid against hand-edited artifacts in practice, the agentic fallback is on the ROADMAP Deferred list.
- **★ The companion is a Claude Code skill, not a host command pair.** `templates/dispatcher/.claude/skills/amanuensis-review/SKILL.md`, installed by `install.sh`. Rationale: owner decision — conversational activation ("let's review the metaphors") over host parity; OpenCode parity is on the Deferred list. Rejected alternative: a `revise`-style command pair on both hosts. The not-a-pipeline-step rule from `revise` still governs: no `step_id`, no recipe line, never touches `pipeline-state.md`.
- **`agents/review-grammars.yaml` is the single source, with adoption markers.** All four grammars are defined now (so M11–M13 implement against a stable contract), but each carries `adopted` / `pending`: a `pending` family's current in-step grammar remains authoritative until its milestone flips the marker. This keeps the YAML honest while the migration is partial. The validator and companion apply only to `adopted` families.
- **Review item shape.** The anchor line `<!-- review-id: ... -->` sits immediately above the item it addresses; `- Decision:` / `- Decision-note:` are nested fields of that item; a blank `Decision:` means pending (per design principle 4; anti-AI's bulk inheritance, M11, is the only exception). `Decision:` holds a legal token (plus payload where the grammar requires one — the machine-readable *what*); `Decision-note:` is optional free text (rationale or clarification — the human *why*). One line = one fact, so the script's line-oriented parsing works.
- **`review-id` grammar.** `<artifact-family>:<location...>:<item-id>`, where the location segments are derivable from the artifact's resolved path and identifiers already present in the report (scene ids, block numbers) — per-`project_type` resolution is defined in the YAML (Task 1 settles the exact literals; short_story collapses the book/chapter segments). IDs are unique within the artifact + reviewed-draft epoch only, and are regenerated with the report on the existing discard-on-regenerate path (ROADMAP shared-design principle 9). Nothing is invented that isn't already on disk.
- **Compliance tokens are unchanged; their *carrier* changes.** `FIX` / `FIX: <instruction>` / `SKIP` / `ESCALATE` move from positional inline annotations into the `Decision:` field. Big-bang: after this Sprint the positional form is an invalid input to `compliance_fix`. `CLEAN` blocks are not review units — no anchor, no fields, no count inflation.
- **The per-unit pending gate is deliberately stronger than today's.** Today `compliance_fix` blocks only when *no* annotations exist and silently skips bare entries (`agents/steps/compliance-fix.md:81`, `:140`); under the contract, *any* actionable violation with a blank `Decision:` is a pending unit and blocks the step. Rationale: silently skipped units are exactly the accepted-vs-unreviewed ambiguity the milestone kills; a human who wants to defer a unit records `SKIP`.
- **Artifact-state mechanics are untouched.** Freshness stays a derived predicate; override stays stale-axis-only, anchor-gated, recorded in the apply log with the existing `Override:` block grammar (`agents/steps/compliance-fix.md:106-132`). For a migrated family, the review evidence M9 calls "annotation" takes the structured form — filled `Decision:` fields — which is a wording generalization in one place (`agents/orchestrator.md`, Task 1), not a model change.
- **Bare `REPLACE` is invalid — locked now, lands behaviorally in M13.** The grammar file records the policy (payload required; `WORKSHOP` is the ask-for-candidates path), superseding the treat-as-`WORKSHOP` convenience at `agents/steps/metaphor-fix.md:69` when M13 retargets the metaphor steps. Rationale: the ROADMAP's stated preferred policy; an implicit normalization is a positional-convention smell.
