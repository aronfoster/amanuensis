# Sprint 16 — Milestone 11: Agent-addressable review — anti-AI slice

This Sprint migrates the **anti-AI family** onto the shared review contract M10 landed, and simplifies it under an owner directive recorded this Sprint: the review process is now **AI-plus-human** — the `amanuensis-review` companion captures and writes, the human decides — so conventions that existed only to make human hand-editing of markdown cheap are retired when retiring them makes agent review simpler, more straightforward, or more reliable. Concretely: `anti_ai_report` emits structured review items (a `review-id` anchor plus blank `- Decision:` / `- Decision-note:` fields per flagged instance) and **drops the `BULK eligibility:` block and the per-scene `### Summary` tally**; the per-category `BULK:` header grammar and bulk inheritance are **retired, not migrated** — category-level review survives as **companion fan-out** (one stated decision written into every pending unit of a declared category, each with an audit note); `anti_ai_fix` consumes pure per-unit decisions through the shared validator, exactly like `compliance_fix`; and the `anti_ai` family flips to `adopted` in `agents/review-grammars.yaml` with its container shape settled so a pre-M11 report can never validate as proceed. `scripts/validate-review-artifact.sh` ships **byte-for-byte unchanged** — verified at planning. This is the second of the four one-milestone-per-sprint slices (M10–M13 in ROADMAP.md; the shared design lives in the completed M10 section there, with principles 5 and 7 revised by this Sprint's planning). Big-bang, as with compliance: after this Sprint the old positional anti-AI annotation format — and any `BULK:` header — is an invalid input to `anti_ai_fix` and no compatibility path is kept.

## Background — what is and isn't wrong today

Established by inspection during planning; tasks should not re-derive this.

- **★ The owner directive that reshaped this milestone.** The anti-AI annotation conventions were designed for a human hand-editing markdown; review is now AI-plus-human, so those conventions should be changed where that makes agent review easier or more reliable (owner, Sprint 16 planning). The ROADMAP M11 section and shared-design principles 5 and 7 were rewritten at planning to match; this SPRINT implements the reshaped milestone.
- **Anti-AI annotation is positional today, and its bulk machinery is hand-edit ergonomics.** Per-entry tokens are written after the quote (`agents/steps/anti-ai-report.md:279-284`); a per-category `BULK: <action>[: <instruction>]` header at the head of the category subsection (`:286-301`) spares the human annotating dozens of entries by hand, gated by the per-scene `BULK eligibility:` block (`:49-65`). That block is **fake-dynamic**: the report emits the same fixed ten-line list every run — it restates static category knowledge that belongs in the grammar file. Header bulk is also the design's *only* blank-means-decided exception and its only two-layer validation case. With the companion as the write path, none of it earns its complexity: the human's "fix all em dashes" can be fanned out into per-unit `Decision:` fields, leaving the artifact grammar identical in shape to compliance's.
- **The per-scene `### Summary` tally (`agents/steps/anti-ai-report.md:255-271`) is a human-reader convenience** the validator's ledger supersedes (and, unlike the ledger, it can drift after appends). Dropping it also removes the only legitimately unit-free `### ` heading in the artifact, which lets the plain `container_pattern` mechanism work with no validator change (see next bullet).
- **The validator needs no change — verified at planning with the stock script and a scratch grammar copy** (anti_ai `adopted`, `bulk_supported: no`, `container_pattern: "### "`, bulk header keys removed): a fan-out-shaped artifact yields the correct ledger (decided/skipped/pending counted per unit, no findings, exit 4 with one blank remaining); a pre-M11 positional report fails structural validation as unanchored containers — including its `### Summary` heading — exit 3, so the old format can never pass as zero-units/proceed (the invariant M10's compliance container check established); and a stray legacy `BULK:` header is rejected as invalid through the script's existing no-bulk-support path (`scripts/validate-review-artifact.sh:370-373`), never silently tolerated. The script's bulk-inheritance machinery and the ledger's `inherited-by-bulk` row become inert (always 0) but stay untouched.
- **The fix step's current gates are weaker than the contract's.** `anti_ai_fix` resolves effective annotations positionally (`agents/steps/anti-ai-fix.md:47-53`), **tolerates an illegal bulk header** by treating it as absent (`:53`), and blocks as `review_pending` only when *no* annotations and *no* bulk headers exist (`:34`, `:193`). Under the contract: every unit needs a filled `Decision:`; any blank is a pending unit and blocks (per-unit gate, compliance model `agents/steps/compliance-fix.md:141`); a `BULK:` header or positional annotation is invalid input and blocks (`:142` model).
- **The contract in `agents/review-grammars.yaml:143-174` predates the reshape** — its `anti_ai` block still defines header bulk (`bulk_supported: yes`, `bulk_actions`, inheritance in `blank_means`/`counting`) and is marked `adoption: pending` / `migrates_in: M11`. Task 1 rewrites it. The file-header comment's "anti-AI bulk inheritance is the only exception" (`:58-59`) and the container-settlement promise (`:38-47`) need the matching updates. `agents/review-validation.md:102-125` (decision automation) describes header-bulk allowances and must be rewritten to the fan-out rules.
- **One category emits two top-level lines per instance.** Synonym Cycling's format is `- Passage: …` plus a sibling `- Cycled terms: …` (`agents/steps/anti-ai-report.md:177-181`). The parser attributes fields to a unit via its single anchored item line (`item_line_pattern: "^- "`, anchor-adjacency rule), so the retargeted format nests `Cycled terms` under the `Passage` line. Every other category already emits one line per instance.
- **Smoke recipes 6–8 hand-author old-format `anti-ai.md` fixtures** (`examples/smoke/README.md:237-395`) and become invalid specimens. Recipes 9–10 (compliance, structured, validator-verdict expectations, companion session) are the model. Compliance is also the model for both step retargets: `compliance_report`'s append-path ordinal continuation (`agents/steps/compliance-report.md:41`) and `compliance_fix`'s validator invocation and per-unit gates (`agents/steps/compliance-fix.md:41-47`, `:141-142`).
- **The step set is unchanged.** No step is added or removed and no `step_id` changes, so `scripts/check-pipeline-state.sh`, both CI workflow ymls, and both `pipeline-state.md` files stay byte-for-byte unchanged.

## Conventions adopted by this Sprint

Locked at planning (starred items are owner decisions from this Sprint's planning session); tasks don't rediscover them.

- **★ The owner directive: review is AI-plus-human; hand-edit ergonomics conventions are retired where that makes agent review simpler or more reliable.** The companion (plus the fix/apply steps) is the primary write path over review artifacts; the human decides, the agent writes. Structured markdown remains the artifact (shared-design principle 8 — readable, diff-friendly, hand-editable in a pinch), but the format no longer optimizes for hand-annotation keystrokes. This directive reshaped the milestone; ROADMAP shared-design principles 5 and 7 were revised at planning.
- **★ Header bulk is retired; category-level review is companion fan-out.** No `BULK:` headers, no `BULK eligibility:` block, no inheritance: where `fanout_categories` in the grammar file declares a category eligible, the human states one decision for the category and the companion writes it into every pending unit's `Decision:` field, marking each `Decision-note:` as a category decision (the audit substitute for the lost explicit-vs-inherited distinction). Rationale: header bulk existed to save hand-annotation keystrokes an agent doesn't have; it was the design's only blank-means-decided exception and only two-layer validation case, and removing it makes blank-means-pending universal, the fix step a pure compliance clone, and the validator untouched. *This supersedes this session's earlier plan revision in which the companion would write `BULK:` headers.* Rejected alternatives: migrating the header grammar as the ROADMAP originally specified (carries the exception and the two-layer machinery forever); keeping the eligibility block as a per-report dynamic layer (it is fake-dynamic — a fixed list emitted every run, restating static knowledge the grammar file owns; single-sourcing puts it in the YAML).
- **★ The bare-`FIX` fallback stays fixer-level; the companion prompts for payloads at capture time.** Categories with no bare-FIX rule (`agents/steps/anti-ai-fix.md:83-90`) keep treating a bare `FIX` as `ESCALATE`, recorded in the apply log; the validator stays category-agnostic with family-level token rules. New under the directive: the skill knows those categories and asks the human for the instruction when capturing `FIX` there, so the fallback rarely fires. Rejected alternative: per-category payload-required rules in the YAML and validator.
- **★ The progress ledger is unchanged, and now the whole validator is too.** `inherited-by-bulk` stays a printed row, permanently 0 (no adopted family grants artifact-level bulk); the script's bulk-parsing paths stay as the rejection mechanism for stray headers. Rationale: the counts and verdicts are the things that must never be wrong; the M10-locked script ships byte-for-byte. Rejected alternative: stripping the inert bulk machinery and ledger row (touches the locked script and every fixture expectation for cosmetic gain; revisit if M12–M13 confirm no family ever uses it).
- **The per-scene `### Summary` tally is dropped.** The validator's ledger is the authoritative count and cannot drift; per-category presentation is the companion's `show category summary`. Dropping it also makes every `### ` heading in the artifact a category container, so `container_pattern: "### "` needs no exemption mechanism — the validator ships unchanged. Rejected alternative: keeping the tally demoted to a non-heading line, or keeping the heading and adding a `container_exempt_prefix` script extension (this Sprint's earlier plan — the tally isn't worth a change to the locked script).
- **A clean scene records `No flags.`** — the scene header plus one line, no anchor, not a review unit (compliance-`CLEAN`'s analog: the audit record that the scene was scanned without inflating counts).
- **One top-level `- ` line per review unit.** The anchor sits immediately above the unit's single item line; auxiliary lines and the decision fields nest below it. Synonym Cycling is the only category this changes: `- Cycled terms:` nests under the `- Passage:` line.
- **Blank means pending, everywhere, with no exceptions.** The report emits blank `Decision:` / `Decision-note:` for every flagged instance; fan-out fills fields at capture time, so the artifact never has a decided-but-blank unit. This retires the last exception to shared-design principle 4.
- **A `BULK:` header and the positional format are invalid input — never silently tolerated.** Replaces `anti_ai_fix`'s treat-as-absent rule (`agents/steps/anti-ai-fix.md:53`): a silently ignored header means the human believes a category is adjudicated when it is not. The validator already rejects both (no-bulk-support path; container check); the fix step blocks on exit 3 naming the findings, per the compliance model.
- **The per-unit pending gate replaces the whole-file gate,** exactly as compliance's did: any flagged instance with a blank `Decision:` is a pending unit and blocks `anti_ai_fix` as `review_pending`, naming the pending review-ids. Deliberately stronger than today's all-bare-only gate (`agents/steps/anti-ai-fix.md:193`), for the reason recorded in Sprint 15: silently skipped units are the accepted-vs-unreviewed ambiguity the milestone kills.
- **Per-entry tokens are unchanged; their *carrier* changes.** `FIX` / `FIX: <instruction>` / `SKIP` / `ESCALATE` move into the `Decision:` field. Big-bang: after this Sprint the positional form is an invalid input to `anti_ai_fix`.
- **Artifact-state mechanics are untouched.** Freshness stays a derived predicate; override stays stale-axis-only, anchor-gated, recorded in the apply log with the existing `Override:` block grammar (`agents/steps/anti-ai-fix.md:159-185`); `agents/orchestrator.md` is not edited this Sprint.
